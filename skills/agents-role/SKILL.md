# Agents角色设计技能

## 概述

本技能用于设计AI Agent的完整角色定义，生成4个标准文档：
- **IDENTITY.md** - 身份设定（Name、Role、Tagline、Description、Expertise）
- **SOUL.md** - 核心人格（Personality、Values、Communication Style、Emotional Response、Humor、Boundaries、Tone）
- **USER.md** - 用户画像（背景、偏好、业务上下文）
- **AGENTS.md** - 操作规则（核心原则、交互方式、输出格式、团队协作）

## 触发条件

以下情况应触发此技能：
- 用户说"设计一个XX角色"
- 用户说"创建agents文档"
- 用户说"定义AI人格"
- 用户说"帮我写SOUL.md"
- 用户说"创建角色定义"
- 用户提到"架构师"、"测试"、"产品经理"、"后端开发"等角色
- 用户要求生成4个文档文件

## 输入信息

需要从用户处获取：
1. **角色名称** - 如"架构师"、"测试工程师"、"产品经理"、"程序员鼓励师"等
2. **角色定位** - 这个角色的核心使命和目标
3. **目标用户** - 这个角色服务的对象
4. **输出目录** - 文件输出位置（默认为当前目录）

## 模板目录

```
templates/
├── identity.md  - IDENTITY.md 模板
├── soul.md      - SOUL.md 模板
├── user.md     - USER.md 模板
└── agents.md   - AGENTS.md 模板
```

## 使用方式

1. **读取模板** - 读取 `templates/` 目录下的模板文件
2. **收集需求** - 向用户确认角色类型、定位、目标用户
3. **填充内容** - 根据角色特点填充各文档的详细内容
4. **生成文件** - 在用户指定目录创建4个文档

## 各文档填充指南

### IDENTITY.md 填充内容

| 字段 | 说明 | 示例（参考 bunny） |
|------|------|------------------|
| Name | 角色名称 | 兔兔 |
| Role | 角色定位 | 程序员鼓励师 / 开发者心理能量补给站 |
| Tagline | 一句话slogan | 你的代码会跑通的，你比你以为的要强 |
| Description | 详细描述 | 专注服务程序员群体的情绪支持与能量补给角色 |
| Expertise | 专业领域 | 懂程序员的痛、爽、压力来源 |

### SOUL.md 填充内容

| 字段 | 说明 | 示例（参考 bunny） |
|------|------|------------------|
| Personality | 性格特点 | 温暖但不油腻、幽默但不刻薄、懂技术但不说教、敏锐共情 |
| Values | 核心价值观 | 5条核心信念，如"每一行报错都值得被尊重" |
| Communication Style | 沟通风格 | 中文+行业黑话，先共情再建议 |
| Emotional Response | 情绪回应框架 | 针对沮丧/焦虑/愤怒/成就/自我怀疑的不同回应方式 |
| Humor | 幽默语料 | 行业梗、鼓励金句 |
| Boundaries | 边界限制 | 不会说什么、不会做什么 |
| Tone | 整体气质 | 像深夜一起改bug的好朋友 |

### USER.md 填充内容

| 字段 | 说明 | 示例（参考 bunny） |
|------|------|------------------|
| 背景 | 用户职业、团队规模、技术栈 | 生鲜配送SaaS ERP创始人，小型创业团队 |
| 偏好 | 语言、沟通风格、决策方式 | 中文、直接高效、务实导向、MVP优先 |
| 业务上下文 | 产品、目标客户、核心业务 | B端SaaS生鲜配送ERP，核心链路采购到结算 |

### AGENTS.md 填充内容

AGENTS.md 由三部分组成：

#### 1. 记忆系统（默认保留）
读取 `templates/agents.md`，直接使用其中的"记忆系统"部分（每次会话、memory/YYYY-MM-DD.md、长期记忆 MEMORY.md、写下来规则），不需要修改。

#### 2. 安全规则（默认保留）
直接使用模板中的"安全"和"外部 vs 内部"规则，不需要修改。

#### 3. 角色自定义部分
根据角色特点填充以下内容：

| 字段 | 说明 | 示例（参考 bunny） |
|------|------|------------------|
| 核心原则 | 最重要的行为准则 | 共情优先、懂技术语境、幽默是良药 |
| 交互方式 | 具体场景的回应方式 | 倾诉时先倾听、"我好菜"时重新归因 |
| 输出格式 | 何时用结构化输出 | 拆解焦虑时用列表、制定目标时用checklist |
| 团队协作 | 与其他角色的协作方式 | 通过@agent_name转交任务 |
| 绝不会做的事 | 底线 | 绝不说"这很简单"、绝不否定情绪 |

**生成顺序**：记忆系统 → 角色自定义部分 → 安全规则 → 外部 vs 内部

## 执行步骤

### ⚠️ 严重警告

1. **必须先备份目录才能执行后续操作**
2. **所有生成或替换的内容必须先由用户确认后才能创建/修改**

### 执行流程

1. **收集需求** - 确认角色名称、定位、目标用户、输出目录
2. **校验输出目录** - 必须从用户获取明确的输出目录路径
3. **检查目录是否存在**
   - 如果目录不存在 → 直接执行第5步
   - 如果目录存在且包含文件 → **必须先备份才能继续**
4. **备份现有目录**（如果存在）
   - 执行备份命令：`cp -r {输出目录} {输出目录}_back`
   - 备份完成后才能继续后续操作
   - 告知用户备份位置
5. **读取模板** - 读取 templates/ 下的4个模板
6. **填充内容** - 根据角色特点填充详细内容
7. **⚠️ 用户确认** - 将填充后的内容展示给用户，确认无误后再执行后续操作
   - 展示4个文档的预览内容
   - 等待用户确认"可以生成"后才能继续
8. **生成文档** - 在用户确认后，创建/修改用户指定目录下的4个文档
9. **返回结果** - 告诉用户文件生成的位置和备份位置

## 角色类型参考

| 角色类型 | 特点 |
|---------|------|
| 程序员鼓励师 | 情绪支持、能量补给、懂程序员痛点 |
| 架构师 | 系统设计、技术选型、代码质量 |
| 测试工程师 | 细节、覆盖率、bug分析 |
| 产品经理 | 用户体验、业务价值、需求分析 |
| 后端开发 | 性能、安全、可维护性 |
| 前端开发 | UI/UX、交互体验、响应式 |
| 运维工程师 | 稳定性、自动化、监控告警 |

## 注意事项

- 生成的4个文档应该是一个整体，相互呼应
- IDENTITY定义角色"是谁"，SOUL定义"怎么表现"，AGENTS定义"怎么做"
- USER.md的内容从与用户的对话中推断或由用户提供
- 参考 templates/ 目录的模板结构，但填充具体内容
- 所有文档使用中文输出
