# Agents角色设计技能

## 概述

本技能用于设计AI Agent的完整角色定义，生成4个标准文档：
- **AGENTS.md** - 定义AI执行任务的具体规则和流程
- **SOUL.md** - 定义AI的核心人格、语气风格、思维模式
- **USER.md** - 让AI真正认识用户
- **IDENTITY.md** - 助手身份设定

## 触发条件

以下情况应触发此技能：
- 用户说"设计一个XX角色"
- 用户说"创建agents文档"
- 用户说"定义AI人格"
- 用户说"帮我写SOUL.md"
- 用户说"创建角色定义"
- 用户提到"架构师"、"测试"、"产品经理"、"后端开发"等角色
- 用户要求生成4个文档文件

## 输入信息

需要从用户处获取：
1. **角色名称** - 如"架构师"、"测试工程师"、"产品经理"等
2. **角色目标** - 这个角色要完成什么任务
3. **用户自定义目录** - 文件输出位置（默认为当前目录下的角色名文件夹）

## 输出结构

在用户指定目录下创建 `{角色名}/` 文件夹，包含以下4个文件。模板存放在 `templates/` 目录下：

### 模板目录

```
templates/
├── identity.md  - IDENTITY.md 模板
├── soul.md      - SOUL.md 模板
├── user.md     - USER.md 模板
└── agents.md   - AGENTS.md 模板
```

### 使用方式

1. 读取 `templates/` 目录下的模板文件
2. 根据用户需求填充模板占位符
3. 将生成的文档写入用户指定目录

## 执行步骤

1. **收集需求** - 向用户确认角色类型和目标
2. **生成文档** - 在用户指定目录创建4个标准文档
3. **填充内容** - 根据角色类型填充符合该角色特点的内容
4. **返回结果** - 告诉用户文件生成的位置

## 角色类型参考

| 角色类型 | 特点 |
|---------|------|
| 架构师 | 关注系统设计、技术选型、代码质量 |
| 测试工程师 | 注重细节、追求完美、重视覆盖率 |
| 产品经理 | 关注用户体验、业务价值、需求分析 |
| 后端开发 | 注重性能、安全、可维护性 |
| 前端开发 | 关注UI/UX、交互体验、响应式设计 |
| 运维工程师 | 注重稳定性、自动化、监控告警 |

## 注意事项

- 生成的文档应该是一个整体，4个文件相互呼应
- SOUL.md中的人格设定应该与AGENTS.md的规则相匹配
- USER.md的内容可以从与用户的对话中推断
- 所有文档使用中文输出，代码示例使用英文
- 保持专业但友好的语气
